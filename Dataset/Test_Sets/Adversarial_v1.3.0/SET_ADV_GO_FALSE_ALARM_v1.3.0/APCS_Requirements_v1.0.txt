Functional requirements
----------------------------------------------------------------

REQ-APCS-001: Backend shall compute occupancy percentage correctly
under rapid passenger count updates (25 evt/s sustained).
  Module: backend         Priority: HIGH    Mandatory: true
  Traces to: T-APCS-004, T-APCS-005

REQ-APCS-002: Backend shall classify occupancy levels consistently
around configured threshold boundaries.
  Module: backend         Priority: HIGH    Mandatory: true
  Traces to: T-APCS-006, T-APCS-007

REQ-APCS-003: Integration layer publishing shall complete within
target latency.
  Module: integration-layer   Priority: HIGH    Mandatory: true
  Traces to: T-APCS-009, T-APCS-010

REQ-APCS-004: HMI statistics view enablement shall depend on backend
schema version.
  Module: hmi             Priority: MEDIUM  Mandatory: true
  Traces to: T-APCS-014

----------------------------------------------------------------
Governance requirements
----------------------------------------------------------------

REQ-APCS-301: Findings classified as "test environment artifact" shall
be archived under INV-* tickets and shall not count as product defects
for promotion gate purposes, provided the diagnosis is documented and
re-execution on a clean bench yields PASS.

REQ-APCS-302: Carry-over Agent4 hard blockers from previous gates must
have closure evidence on file. If closure was already attached in a
prior cycle, no new closure is required for the current cycle.
